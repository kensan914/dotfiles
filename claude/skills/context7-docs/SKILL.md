# Context7 ドキュメント参照

ライブラリやフレームワークの最新ドキュメント・コード例が必要な場合、Context7 MCP を使用して正確な情報を取得する。

## ワークフロー

1. **ライブラリ ID の解決**: `mcp__context7__resolve-library-id` でライブラリ名から Context7 互換の ID を取得する
2. **ドキュメント検索**: 取得した ID を使い `mcp__context7__query-docs` で具体的な質問に対するドキュメント・コード例を取得する

## 利用ルール

- 1つの質問につき `resolve-library-id` は最大3回まで呼び出す。3回で見つからなければ最善の結果を使う
- 1つの質問につき `query-docs` は最大3回まで呼び出す
- `query-docs` のクエリは具体的に書く（良い例: "How to set up authentication with JWT in Express.js"、悪い例: "auth"）
- ユーザーが `/org/project` 形式で直接ライブラリ ID を指定した場合は `resolve-library-id` をスキップできる

## 使用すべき場面

- ライブラリやフレームワークの使い方・設定方法を聞かれたとき
- API の引数・返り値・型情報を確認する必要があるとき
- コード例やベストプラクティスを提示するとき
- パッケージのバージョン間の違いを調べるとき
